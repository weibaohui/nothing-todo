# Atomcode 输出样本

Atomcode 的输出比较特殊——**不是 JSON 格式**。stdout 是纯文本，stderr 包含结构化事件。

## 输出结构

| 流 | 格式 | 含义 |
|----|------|------|
| stdout | 纯文本 | AI 回复内容（直接透传） |
| stderr | `[tokens] prompt=N completion=M` | Token 统计 |
| stderr | `[tool→ <name> args={<json>}]` | 工具调用发起 |
| stderr | `[tool← <name> args={<json>}]` | 工具调用返回 |
| stderr | `[done] <duration> tokens=N turns=N tool_calls=N` | 执行完成 |
| stderr | `[tool-streaming...]` | 跳过（流式标记） |
| stderr | `[headless]` | 跳过（headless 标记） |

## stdout 样本（纯文本）

```
我来分析一下这个问题。

首先，让我看看当前目录下的文件结构。
```

## stderr 样本（结构化事件）

```
[tool→ bash args={"command": "ls -la"}
[tool← bash args={"command": "ls -la"}
[tokens] prompt=120 completion=45
[tool→ Read args={"file_path": "src/main.rs"}
[tool← Read args={"file_path": "src/main.rs"}
[tokens] prompt=320 completion=120
[done] 4.6s tokens=100 turns=2 tool_calls=1
```
